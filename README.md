# ozon-reviews-parser

CLI-утилита для сбора отзывов о товарах [Ozon](https://www.ozon.ru) в JSON-файлы.
Один товар — один файл `output/{product_id}.json`. Задумана как источник данных для анализа отзывов человеком или ИИ.

## Возможности

- Сбор отзывов по ссылке на товар (короткой `ozon.ru/t/...` или полной) либо по списку ссылок из файла.
- Фильтр по актуальности: только отзывы за последний год (период настраивается).
- Фильтр по варианту: по умолчанию — все варианты товара; `--this-variant` — только вариант из ссылки.
- Карточка товара: название, цена (обычная / с Ozon-картой / старая), полные характеристики.
- Подробные поля отзыва: автор, оценка, дата, текст, достоинства, недостатки, лайки/дизлайки, факт покупки, ссылки на фото и видео, вариант.
- Вопросы и ответы: только отвеченные вопросы, со всеми ответами (магазин, бренд и пользователи).
- Обход анти-бота Ozon через [Patchright](https://github.com/Kaliiiiiiiiii-Vinyzu/patchright-python) (реальный Chrome без следов автоматизации), без входа в аккаунт.

## Требования

- Windows, Python 3.11+
- Установленный **Google Chrome** (используется реальный браузер, отдельный не скачивается)
- **VPN выключен** — Ozon блокирует VPN/прокси-адреса

## Установка

```powershell
py -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## Использование

```powershell
# Все варианты, отзывы за год (по умолчанию)
.\.venv\Scripts\python.exe main.py "https://ozon.ru/t/xxxxxxx"

# Только вариант из ссылки (когда под карточкой собраны разные товары)
.\.venv\Scripts\python.exe main.py "<ссылка>" --this-variant

# Список ссылок из файла (по одной в строке, # — комментарий)
.\.venv\Scripts\python.exe main.py -f urls.txt
```

| Флаг | Назначение |
|---|---|
| `--this-variant` | только вариант из ссылки (по умолчанию — все варианты) |
| `--years N` | период актуальности отзывов в годах (по умолчанию 1) |
| `--max N` | максимум отзывов на товар (по умолчанию 500) |
| `--headless` | без окна браузера (на Ozon обычно блокируется) |

Окно Chrome по умолчанию видимое — это необходимо: в headless-режиме Ozon обычно показывает страницу проверки.

Блок `stats` — сводка оценок (окна: 30/90/180/365 дней):
- `overall` — средняя и общее число отзывов **от Ozon** (истина за всё время; анонимная выборка для этого не годится).
- `collected` — сколько собрано: всего / с текстом / пустых.
- `windows` — средняя, распределение по звёздам и `count` по **собранным** отзывам, включая пустые (у них есть оценка). `covered: false` означает, что выборка не покрывает всё окно (нет отзывов старше его начала) — цифры смещены к свежим, доверять им меньше.

## Как это работает

1. Запускается реальный Chrome через Patchright (`channel="chrome"`), без входа в аккаунт.
2. Браузер проходит проверку Ozon; сессия сохраняется в `.profile` для повторных запусков.
3. Из живой сессии берутся заголовки внутреннего API; лента отзывов страницы `/reviews/` запрашивается через `fetch` по курсору `nextPage` (по 30 за запрос) в трёх сортировках — новые, низкая и высокая оценка — для полного охвата и баланса позитива/негатива.
4. Отзывы фильтруются по дате и варианту (`itemId`), пустые (без текста) пропускаются, сортируются «сначала новые».
5. Цена и характеристики берутся с карточки и страницы `/features/`; вопросы — со страницы `/questions/` (сорт «сначала с ответом»), а все ответы многоответных вопросов догружаются со страницы `/question/<id>/`.
6. Результат сохраняется в `output/{product_id}.json`.

`scripts/recon.py` — диагностический скрипт: сохраняет сырые ответы Ozon в `captures/` (полезно при поломке парсера после изменений на сайте).

## Глубина и лимиты (без аккаунта)

Проверено `scripts/audit.py` на товаре с 5066 отзывами:

- Прямой `fetch`-курсор **обходит** UI-стену «Войдите»: вручную в браузере лента упирается через пару месяцев, а `fetch` уходит на ~3.5 года назад.
- Анонимно каждая сортировка отдаёт ~990 отзывов и честно заканчивается (курсор иссякает) — предел не в `fetch`, а в самом API Ozon.
- Три сортировки дают разные срезы: в сумме ~2526 уникальных из 5066 (≈50%). Полный объём доступен только под аккаунтом (не реализовано).
- Около 60% отзывов — пустые (только звезда, без текста), они отсеиваются.
- В пределах года это давало ~320 текстовых отзывов против ~209 у одной хронологической ленты — поэтому добор по сортировкам включён по умолчанию.
- Данные уходят на годы назад, так что для малопопулярных товаров можно ставить `--years 2`/`3`.
- Скорость: сбор идёт прямыми запросами, весь товар — несколько секунд (время выводится в консоль).

Свой аудит: `.\.venv\Scripts\python.exe scripts/audit.py "<ссылка>" --all-sorts`.

## Ограничения

- Без аккаунта доступно ~90 вопросов на товар (сорт «сначала с ответом» → берутся отвеченные) и ~2526 из 5066 отзывов — см. «Глубина и лимиты».
- Автоматический сбор нарушает правила использования Ozon. Используйте для личных целей и с разумными паузами.
- Парсер зависит от текущей структуры Ozon — при изменениях на стороне сайта потребуется правка.

## Планы развития

Базовый сбор (текущий `main`) самостоятелен и работает без расширений. Новые возможности добавляются **опционально** — отдельными флагами и зависимостями, чтобы тем, кому нужны только данные по товару, ничего лишнего не требовалось. Каждая возможность разрабатывается в своей ветке.

- **Статистика отзывов** — распределение оценок по окнам (30/90/365 дней) рядом со средним за всё время: видно, как меняется качество со временем (свежие отзывы часто ниже общей оценки).
- **Самопроверка парсинга** — офлайн-регрессия на реальных ответах Ozon + режим `--doctor`, который сразу показывает, какой блок перестал отдаваться (вместо молчаливого пустого JSON).
- **ИИ-анализ (опционально)** — вердикт по собранным данным: локальной моделью через Ollama или по API-ключу. Включается флагом, на базовый режим не влияет.
- **Локальный веб-интерфейс** — вставил ссылку, получил файл, статистику и вердикт, без командной строки (только на своей машине: нужен видимый Chrome и выключенный VPN).

## Если Ozon начинает блокировать

Убедитесь, что окно видимое (не `--headless`) и VPN выключен. Более незаметный фолбэк без переписывания логики — [Camoufox](https://github.com/daijro/camoufox) (управляется тем же Playwright API).
